🌎 1. What is TravelNest?

TravelNest is a travel planning website built using only HTML, CSS, and JavaScript — no frameworks or libraries. It has 6 pages:

  • Home — rotating quotes, Destination of the Day, features overview
  • Destination Explorer — browse + filter 12 destinations, click for full details
  • Budget Planner — calculate total trip cost with animated results
  • Trip Generator — randomly suggest a destination based on travel style
  • Travel Mood — ambient sounds + mark destinations visited/planned
  • Feedback & Support — validated contact form + FAQ accordion
💡 One-liner answer: "TravelNest is a multi-page front-end website that helps users explore travel destinations, plan their budget, and track their travel goals — all built from scratch with HTML, CSS, and JavaScript."

Destination of the Day

JavaScript works out what day of the year it is, then uses that number to pick a destination. The same destination shows all day — it changes automatically at midnight.

var today = new Date();
var startOfYear = new Date(today.getFullYear(), 0, 1);
var dayOfYear = Math.floor((today - startOfYear) / (1000 * 60 * 60 * 24));
var index = dayOfYear % destinations.length; // % wraps around so it never goes out of bounds
var dest  = destinations[index];

The % symbol is the modulo operator — it gives the remainder after division. So day 365 % 12 destinations = index 5. It always stays within bounds.

💰 9. Trip Budget Planner

Calculation

var total = days * daily; // e.g. 7 days × £80/day = £560 total

Budget status is classified by daily spend:

  • Low — ≤ £50/day
  • Moderate — £51–£150/day
  • Luxury — more than £150/day

Animated Counter

I use requestAnimationFrame — a browser method that calls a function ~60 times per second (in sync with the screen). Each frame I increase the displayed number a little until it reaches the target total.

function animateCounter(element, target, duration) {
    var startTime = null;
    function step(timestamp) {
        var progress = (timestamp - startTime) / duration;
        element.textContent = '£' + Math.floor(progress * target);
        if (progress < 1) requestAnimationFrame(step); // keep going
    }
    requestAnimationFrame(step); // start
}

Progress Bar

The bar is a <div> with width: 0%. I set the width in JavaScript. Because the CSS has transition: width 1.2s ease, it animates smoothly.

fill.style.width = '65%'; // CSS transition handles the animation automatically

🎵 11. Travel Mood Page

Ambient Sounds — Web Audio API

I used the Web Audio API, which is built into all modern browsers. It generates sounds with code — no audio files needed.

The idea: Create random number data (white noise), then filter it to sound like a specific environment.

var audioCtx = new AudioContext();  // the audio engine

// Create a buffer of random numbers = white noise
var buffer = audioCtx.createBuffer(1, audioCtx.sampleRate * 3, audioCtx.sampleRate);
var data = buffer.getChannelData(0);
for (var i = 0; i < data.length; i++) {
    data[i] = Math.random() * 2 - 1; // random values -1 to 1
}

// Connect through a filter to shape the sound
var filter = audioCtx.createBiquadFilter();
filter.type = 'lowpass';      // only lets low frequencies through
filter.frequency.value = 450; // cut-off at 450 Hz

source.connect(filter);
filter.connect(audioCtx.destination); // destination = speakers
source.start();
SoundFilter TypeWhy it Sounds That Way
Beach WavesLow-passOnly low frequencies pass through → rolling wave sound
Forest RainBand-passMid frequencies pass → rain/wind rustling sound
City HumLow-pass + oscillator at 60HzDeep hum like traffic and buildings

Destination Tracking — Visited & Planned

Each destination has two separate buttons: Visited and Planned. They store destination IDs in two separate localStorage arrays: tn_visited and tn_planned.

function toggleTracking(action, id) {
    var key  = action === 'visited' ? 'tn_visited' : 'tn_planned';
    var list = loadFromStorage(key) || [];
    var index = list.indexOf(id);

    if (index === -1) {
        list.push(id);         // not in list — add it
    } else {
        list.splice(index, 1); // already in list — remove it (toggle off)
    }
    saveToStorage(key, list);
}

💾 13. localStorage — What and Why?

localStorage is storage built into the browser that persists after closing the tab. I use it to save user data without a database or server.

localStorage only stores strings, so I convert objects to JSON strings when saving and convert back when loading.

// Saving
localStorage.setItem('tn_budgets', JSON.stringify(budgetsArray));

// Loading
var budgets = JSON.parse(localStorage.getItem('tn_budgets'));

👀 14. Scroll Reveal Animation

I use the IntersectionObserver API. It watches elements and fires a function when they enter the visible area (viewport).

var observer = new IntersectionObserver(function(entries) {
    entries.forEach(function(entry) {
        if (entry.isIntersecting) {          // element is now visible
            entry.target.classList.add('revealed');
            observer.unobserve(entry.target); // stop watching it
        }
    });
}, { threshold: 0.12 }); // trigger when 12% of the element is visible

Elements with class reveal start invisible and shifted down. When revealed is added, the CSS transition animates them into place.

.reveal   { opacity: 0; transform: translateY(28px); transition: opacity 0.6s, transform 0.6s; }
.revealed { opacity: 1; transform: translateY(0); }

📱 16. PWA — Progressive Web App

A PWA means the website can be installed on a phone and work offline. Two files make this work:

manifest.json

Tells the browser the app's name, icon, and how it should look when installed on the home screen.

{
  "name": "TravelNest",
  "short_name": "TravelNest",
  "display": "standalone",   // looks like a native app (no browser bar)
  "start_url": ".",
  "theme_color": "#1e40af",
  "icons": [{ "src": "images/icon.svg", "sizes": "any" }]
}

sw.js — Service Worker

A service worker is a script that runs in the background. On the first visit it caches (saves) all the HTML, CSS, and JS files. On future visits it serves those from the cache — so the site loads fast and works offline.

self.addEventListener('fetch', function(event) {
    event.respondWith(
        caches.match(event.request).then(function(cached) {
            return cached || fetch(event.request); // cache first, then network
        })
    );
});

❓ 18. Quick-Fire Questions & Answers

Click any question to reveal the answer.

Why HTML, CSS, and JavaScript only — no React or Vue? +
The assignment brief specifically says "HTML, CSS, and JavaScript only". No frameworks allowed. This also demonstrates I understand the fundamentals without relying on tools that do the work for me.
What is the difference between var, let, and const? +
var is the old way — it's function-scoped. let is block-scoped (only lives inside its curly braces). const is for values that never change. I mostly used var and let.
What is JSON and why did you use it? +
JSON stands for JavaScript Object Notation. It's a text format for storing structured data. localStorage can only store strings, so I use JSON.stringify() to convert objects to text before saving, and JSON.parse() to convert back when reading.
What does event.preventDefault() do? +
It stops the browser's default action. On a form, the default is to reload the page when submitted. I call event.preventDefault() so I can validate and handle the submission with my own JavaScript instead.
What is Math.random() and how did you use it? +
Math.random() returns a random decimal between 0 and 1. To get a random array index I do: Math.floor(Math.random() * array.length). Math.floor rounds down to the nearest whole number, giving a valid index.
Why does Destination of the Day use modulo (%)? +
There are 12 destinations but up to 365 days in a year. Without modulo, the index would go past the array's end and cause an error. Modulo (%) gives the remainder — so day 13 % 12 = 1, day 365 % 12 = 5. It always wraps around to a valid index.
What is Array.filter() and how did you use it? +
filter() creates a new array containing only the items that pass a test function. The test returns true to keep the item or false to remove it. The original array is untouched. I use it on the Destinations page to filter by continent and name, and in the Generator to filter by travel type and budget.
What is a regular expression (regex) and where did you use it? +
A regex is a pattern used to match text. I used /^[^\s@]+@[^\s@]+\.[^\s@]+$/ to validate email addresses. In plain English: "one or more characters, then @, then one or more characters, then a dot, then one or more characters". If the email doesn't match this pattern, I show an error.
What is requestAnimationFrame and why is it better than setInterval for animation? +
requestAnimationFrame calls a function just before the screen repaints (~60 times/second) and syncs with the display's refresh rate. setInterval doesn't sync with the screen, so it can cause choppy animation. I used it for the animated budget counter.
What is a service worker? +
A service worker is a JavaScript file that runs in the background, separate from the web page. It acts like a middleman between the browser and the network. I used it to cache all the site's files so the site loads fast and works offline. It's what makes TravelNest a PWA.
Why did you separate the JS into multiple files? +
Each file has one clear responsibility. data.js holds the data, utils.js holds shared functions, and each page has its own JS file. This makes the code easier to read, maintain, and debug. If something breaks on the Budget page, I look in budget.js only.
What is the IntersectionObserver API? +
It's a browser API that watches elements and fires a callback when they enter or exit the visible part of the screen. I use it to trigger the scroll reveal animation — when an element scrolls into view, I add the class revealed which triggers the CSS fade-in transition.
What is semantic HTML and why does it matter? +
Semantic HTML uses tags that describe the meaning of content, not just its appearance. For example, <nav> tells the browser "this is navigation" — a <div> with the same CSS would look identical but carry no meaning. It matters for accessibility (screen readers understand the structure) and SEO (search engines rank it better).
How does the Web Audio API create different sounds without audio files? +
I fill a memory buffer with random numbers — this is called white noise. Then I connect it through a BiquadFilter which is like an EQ — it boosts or cuts certain frequency ranges. Low-pass only lets low frequencies through (sounds like waves). Band-pass keeps mid frequencies (sounds like rain). Adding a 60Hz oscillator creates a deep hum for city sounds. No files needed — it's all maths.
